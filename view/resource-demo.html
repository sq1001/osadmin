资源加载方式示例

本页面演示三种资源加载方式的使用方法

推荐 方式一：配置文件方式

在 config/resources.json 中声明页面依赖

config/resources.json 
{
  "pages": {
    "view/resource-demo.html": {
      "css": ["admin/css/view/resource-demo.css"],
      "js": ["admin/js/view/resource-demo.js"],
      "dependencies": ["echarts"]
    }
  },
  "externals": {
    "echarts": {
      "js": "lib/echarts/echarts.min.js",
      "css": null,
      "global": "echarts"
    }
  }
}
✅

优点

  • 配置集中管理，便于维护
  • 支持依赖关系管理
  • 资源加载顺序可控
  • 适合大型项目
📋

使用步骤

  1. 在 config/resources.json 中添加页面配置
  2. 声明该页面需要的 CSS 和 JS 文件
  3. 声明第三方依赖（如 echarts）
  4. 系统会在加载页面时自动加载这些资源

简单 方式二：页面内联方式

在页面 HTML 的 head 中直接声明资源

view/your-page.html 
<!DOCTYPE html>
<html>
<head>
  <!-- ✅ 这个 CSS 会被自动加载 -->
  <link rel="stylesheet" href="admin/css/your-page.css">
  
  <!-- ✅ 这个 JS 会被自动加载 -->
  <script src="admin/js/your-page.js"></script>
  
  <!-- ❌ 注释中的资源会被忽略 -->
  <!-- <link rel="stylesheet" href="deprecated.css"> -->
</head>
<body>
  <div class="page-content">
    页面内容
  </div>
</body>
</html>
✅

优点

  • 简单直观，易于理解
  • 资源与页面绑定，便于查找
  • 适合小型项目或单页面
⚠️

注意事项

  • HTML 注释中的资源会被自动过滤
  • 已加载的资源不会重复加载
  • 不支持依赖关系管理

灵活 方式三：编程方式

在 JavaScript 代码中动态加载资源

JavaScript 代码示例 
// 获取资源加载器实例
var loader = OSLAY.modules.resourceLoader;

// 示例1：加载单个 CSS 文件
loader.loadCSS('admin/css/custom.css')
  .done(function() {
    console.log('CSS 加载成功');
  })
  .fail(function() {
    console.error('CSS 加载失败');
  });

// 示例2：加载单个 JS 文件
loader.loadJS('admin/js/custom.js')
  .done(function() {
    console.log('JS 加载成功');
  })
  .fail(function() {
    console.error('JS 加载失败');
  });

// 示例3：加载第三方依赖
loader.loadDependency('echarts')
  .done(function() {
    console.log('ECharts 加载成功');
    // 现在可以使用 echarts 对象了
    var chart = echarts.init(document.getElementById('chart'));
  });

// 示例4：并行加载多个资源
$.when(
  loader.loadCSS('page1.css'),
  loader.loadCSS('page2.css'),
  loader.loadJS('page1.js'),
  loader.loadJS('page2.js')
).done(function() {
  console.log('所有资源加载完成');
  // 执行初始化代码
  initPage();
});

// 示例5：检查资源是否已加载
if (loader.isLoaded('css', 'admin/css/custom.css')) {
  console.log('该 CSS 已加载');
}

if (loader.isLoaded('js', 'admin/js/custom.js')) {
  console.log('该 JS 已加载');
}
✅

优点

  • 最灵活，可按需动态加载
  • 支持条件加载（根据业务逻辑）
  • 支持懒加载和预加载
  • 可以控制加载时机
🎯

适用场景

  • 用户交互触发的资源加载
  • 条件加载（如权限控制）
  • 懒加载（如滚动到某区域）
  • 插件动态加载

最佳实践建议

如何在实际项目中使用资源加载系统

1

资源分类管理

将资源按类型和用途分类存放：

  • 全局资源：在 index.html 中直接引入（如 layui.css）
  • 页面资源：在 resources.json 中配置（如 product-list.css）
  • 组件资源：使用编程方式懒加载（如富文本编辑器）
2

避免重复加载

系统会自动去重，但建议：

  • 全局资源不要在页面中重复声明
  • 使用 isLoaded() 检查资源状态
  • 第三方库统一在 externals 中配置
3

性能优化

提升页面加载性能的技巧：

  • 合并小文件，减少 HTTP 请求
  • 使用懒加载，延迟非关键资源
  • 预加载关键资源，提升用户体验
  • 压缩 CSS/JS 文件，减小文件大小
4

调试技巧

如何调试资源加载问题：

  • 打开浏览器开发者工具的 Network 面板
  • 查看 Console 中的加载日志
  • 使用 OSLAY.modules.resourceLoader.isLoaded() 检查
  • 检查资源路径是否正确
